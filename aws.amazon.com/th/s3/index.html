Amazon S3

พื้นที่จัดเก็บออบเจ็กต์ได้ถูกสร้างขึ้นเพื่อจัดเก็บและเรียกค้นข้อมูลที่ต้องการได้จากทุกที่

เริ่มต้นใช้งาน Amazon S3
ขอดูข้อมูลเพิ่มเติม

Amazon Simple Storage Service (Amazon S3) คือบริการพื้นที่จัดเก็บอ็อบเจ็กต์ที่มอบการปรับขนาด ความพร้อมใช้งานของข้อมูล ความปลอดภัย และประสิทธิภาพชั้นนำในอุตสาหกรรม หมายความว่าลูกค้าในทุกระดับความต้องการและภาคอุตสาหกรรมจะสามารถนำไปใช้เพื่อจัดเก็บและป้องกันข้อมูลปริมาณใดก็ตามที่มีการใช้งานที่หลากหลาย เช่น เว็บไซต์ แอปพลิเคชันมือถือ การสำรองข้อมูลและการคืนค่า การเก็บถาวร การใช้งานในองค์กร อุปกรณ์ IoT และการวิเคราะห์ Big Data Amazon S3 มอบคุณสมบัติการจัดการที่ใช้งานได้ง่าย เพื่อให้คุณสามารถจัดระเบียบข้อมูลและกำหนดค่าการควบคุมการเข้าถึงที่ตรงกันเพื่อตอบสนองต่อความต้องการของธุรกิจ องค์กร และการปฏิบัติตามข้อกำหนดของคุณ Amazon S3 ได้รับการออกแบบมาเพื่อความทนทาน 99.999999999% (มี 9 ถึง 11 ตัว) และจัดเก็บข้อมูลเพื่อการใช้งานกว่าล้านรายการจากบริษัทรอบโลก

ประโยชน์

ประสิทธิภาพ ความสามารถในการปรับขนาด ความพร้อมใช้งาน และความทนทานชั้นนำแห่งอุตสาหกรรม

ปรับเพิ่มและลดขนาดทรัพยากรพื้นที่จัดเก็บให้ตอบสนองต่อความต้องการ โดยไม่ต้องมีการลงทุนเบื้องตนหรือวงจรการจัดซื้อทรัพยากร Amazon S3 ได้รับการออกแบบมาเพื่อความทนทานของข้อมูล 99.999999999% (มี 9 ถึง 11 ตัว) เพราะมันจะสร้างและจัดเก็บสำเนาของอ็อบเจ็กต์ S3 ทั้งหมดบนระบบต่างๆ ได้โดยอัตโนมัติ หมายความว่า ข้อมูลของคุณจะพร้อมใช้งานเมื่อจำเป็น และจะได้รับการปกป้องจากความล้มเหลว ข้อผิดพลาด และภัยคุกคาม
เรียนรู้เพิ่มเติมเกี่ยวกับความทนทานของข้อมูล S3 »

หลากหลายคลาสพื้นที่จัดเก็บที่คุ้มค่า

ประหยัดค่าใช้จ่ายโดยไม่ต้องยอมลดประสิทธิภาพด้วยการจัดเก็บข้อมูลบนคลาสพื้นที่จัดเก็บ S3 ซึ่งรองรับระดับการเข้าถึงข้อมูลที่หลากหลายไปตามอัตราส่วน คุณสามารถใช้การวิเคราะห์คลาสพื้นที่จัดเก็บ S3 เพื่อค้นพบข้อมูลที่ควรย้ายไปยังคลาสพื้นที่จัดเก็บที่ถูกกว่า อิงตามรูปแบบการเข้าถึง และกำหนดค่านโยบายรอบการใช้งาน S3 เพื่อดำเนินการถ่ายโอน คุณยังสามารถจัดเก็บข้อมูลด้วยรูปแบบการเข้าถึงที่เปลี่ยนแปลงหรือไม่ทราบในการจัดระดับแบบอัจฉริยะของ S3 ซึ่งจะจัดระดับอ็อบเจ็กต์ตามรูปแบบการเข้าถึงที่เปลี่ยนแปลงและมอบการประหยัดค่าใช้จ่ายได้โดยอัตโนมัติ
เรียนรู้เพิ่มเติมเกี่ยวกับคลาสพื้นที่จัดเก็บ S3 »

ความปลอดภัย การปฏิบัติตามข้อกำหนด และความสามารถในการตรวจสอบเหนือคู่แข่ง

จัดเก็บข้อมูลใน Amazon S3 และป้องกันการเข้าถึงที่ไม่ได้รับอนุญาตด้วยคุณสมบัติการเข้ารหัสและเครื่องมือจัดการการเข้าถึง S3 เป็นบริการพื้นที่จัดเก็บวัตถุเพียงแห่งเดียวที่ให้คุณบล็อกการเข้าถึงแบบสาธารณะให้กับวัตถุทั้งหมดของคุณในบัคเก็ต หรือระดับบัญชีที่มีการบล็อกการเข้าถึงแบบสาธารณะของ S3 S3 จะรักษาโปรแกรมการปฏิบัติตามข้อกำหนด เช่น PCI-DSS, HIPAA/HITECH, FedRAMP, Eu Data Protection Directive และ FISMA เพื่อช่วยให้คุณมีคุณสมบัติตรงตามข้อกำหนด AWS ยังสนับสนุนความสามารถในการตรวจสอบที่หลากหลายเพื่อติดตามคำขอการเข้าถึงทรัพยากร S3 ของคุณ
เรียนรู้เพิ่มเติมเกี่ยวกับความปลอดภัยของ S3 และการปฏิบัติตามข้อกำหนด »

จัดการข้อมูลและเข้าถึงการควบคุมได้อย่างง่ายดาย

S3 ทำให้คุณสามารถจัดการการเข้าถึง, ต้นทุน, การจำลอง และการปกป้องข้อมูลอย่างมีประสิทธิภาพ S3 Access Points ทำให้การจัดการการเข้าถึงข้อมูลเป็นเรื่องง่ายด้วยสิทธิ์ที่เฉพาะเจาะจงสำหรับแอปพลิเคชันของคุณโดยใช้ชุดข้อมูลที่แชร์ คุณสมบัติ S3 Replication จัดการการจำลองข้อมูลภายในภูมิภาคหรือภูมิภาคอื่น S3 Batch Operations ช่วยจัดการการเปลี่ยนแปลงขนาดใหญ่ในอ็อบเจ็กต์หลายพันล้านรายการ เนื่องจาก Amazon S3 ทำงานกับ AWS Lambda คุณจึงสามารถบันทึกกิจกรรม กำหนดการแจ้งเตือน และกำหนดเวิร์กโฟลว์อัตโนมัติโดยไม่ต้องจัดการโครงสร้างพื้นฐานเพิ่มเติม
เรียนรู้เพิ่มเติมเกี่ยวกับคุณสมบัติการจัดการพื้นที่จัดเก็บข้อมูล S3 »

บริการสืบค้นในตัวสำหรับการวิเคราะห์

เรียกใช้การวิเคราะห์บนอ็อบเจ็กต์ S3 ของคุณ (รวมถึงชุดข้อมูลอื่นๆ ใน AWS) ด้วยบริการสืบค้นในตัวของเรา ใช้ Amazon Athena เพื่อสืบค้นข้อมูล S3 ด้วยนิพจน์ SQL มาตรฐานและ Amazon Redshift Spectrum เพื่อวิเคราะห์ข้อมูลที่จัดเก็บไว้ในคลังข้อมูล AWS และทรัพยากร S3 ของคุณ คุณยังสามารถใช้ S3 Select เพื่อเรียกดูชุดย่อยของข้อมูลอ็อบเจ็กต์แทนที่จะเป็นอ็อบเจ็กต์ทั้งหมด และพัฒนาประสิทธิภาพการสืบค้นได้ถึง 400%
เรียนรู้เพิ่มเติมเกี่ยวกับการสืบค้นในตัว »

บริการพื้นที่จัดเก็บระบบคลาวด์ที่ได้รับการรองรับมากที่สุด

จัดเก็บและปกป้องข้อมูลของคุณใน Amazon S3 ผ่านการร่วมงานกับคู่ค้าจาก AWS Partner Network (APN) ซึ่งเป็นชุมชนแห่งผู้ให้บริการคำปรึกษาและเทคโนโลยีบนระบบคลาวด์ที่ใหญ่ที่สุด APN จดจำคู่ค้าด้านการโยกย้ายที่โอนถ่ายข้อมูลมายัง Amazon S3 และคู่ค้าด้านพื้นที่จัดเก็บที่มอบโซลูชันการผสานรวมกับ S3 สำหรับพื้นที่จัดเก็บหลัก การสำรองข้อมูลและการกู้คืนข้อมูล การเก็บถาวร และการกู้คืนข้อมูลหลังภัยพิบัติ คุณยังสามารถซื้อโซลูชันการผสานรวมกับ AWS ได้โดยตรงจาก AWS Marketplace ซึ่งมีข้อเสนอด้านพื้นที่จัดเก็บโดยเฉพาะกว่า 250 รายการ
เรียนรู้เกี่ยวกับ APN และ AWS Marketplace »

วิธีการทำงานของ S3 Access Points

Amazon S3 Access Points ช่วยลดความซับซ้อนในการจัดการการเข้าถึงข้อมูลปริมาณมากสำหรับแอปพลิเคชันที่ใช้ชุดข้อมูลที่มีการแชร์บน S3 ด้วย S3 Access Point คุณจะสามารถสร้าง Access Point ได้นับร้อยรายการต่อหนึ่งบัคเก็ต ซึ่งเป็นวิธีใหม่ในการจัดเตรียมการเข้าถึงให้กับชุดข้อมูลที่แชร์ Access Point มีเส้นทางไปยังบัคเก็ตที่กำหนดเอง พร้อมกับชื่อโฮสต์ที่ไม่ซ้ำกันและนโยบายการเข้าถึงที่บังคับใช้สิทธิ์ที่เฉพาะเจาะจงและการควบคุมเครือข่ายสำหรับคำขอทุกรายการที่ทำผ่าน Access Point

กรณีใช้งาน

การสำรองและการกู้คืนข้อมูล

สร้างโซลูชันการสำรองข้อมูลและคืนค่าที่ปรับขนาดได้ ทนทาน และปลอดภัย ด้วย Amazon S3 และบริการอื่นๆ ของ AWS เช่น S3 Glacier, Amazon EFS และ Amazon EBS เพื่อเพิ่มการลงทุนหรือแทนที่ความสามารถในสถานที่ที่มีอยู่แล้ว คู่ค้าของ APN และ AWS จะช่วยให้คุณสามารถตรงตามข้อกำหนดของ Recovery Time Objectives (RTO), Recovery Point Objectives (RPO) และกฎข้อบังคับ ด้วย AWS คุณสามารถสำรองข้อมูลที่มีอยู่ใน AWS Cloud หรือใช้บริการพื้นที่จัดเก็บแบบไฮบริด AWS Storage Gateway เพื่อส่งสำเนาสำรองของข้อมูลในสถานที่ไปยัง AWS
เรียนรู้เพิ่มเติมเกี่ยวกับการสำรองและการกู้คืนข้อมูล »

การกู้คืนข้อมูลหลังภัยพิบัติ (DR)

ป้องกันข้อมูล แอปพลิเคชัน และระบบ IT ที่สำคัญทำงานอยู่บน AWS Cloud หรือในสภาพแวดล้อมสถานที่ทำงานของคุณโดยไม่ต้องเสียค่าใช้จ่ายสำหรับไซต์สำรอง ด้วยพื้นที่จัดเก็บ Amazon S3 การทำซ้ำข้ามภูมิภาคของ S3 และการประมวลผล เครือข่าย และบริการฐานข้อมูลของ AWS คุณจะสามารถสร้างสถาปัตยกรรม DR เพื่อกู้คืนจากความขัดข้องอันเนื่องมาจากภัยพิบัติทางธรรมชาติ ความล้มเหลวของระบบ และความผิดพลาดของมนุษย์ได้อย่างรวดเร็วและง่ายดาย
เรียนรู้เพิ่มเติมเกี่ยวกับ DR »

การเก็บถาวร

เลิกใช้โครงสร้างพื้นฐานทางกายภาพ และเก็บข้อมูลถาวรด้วย S3 Glacier และ S3 Glacier Deep Archive คลาสพื้นที่จัดเก็บ S3 เหล่านี้จะเก็บรักษาอ็อบเจ็กต์ระยะยาวในอัตราที่ต่ำที่สุด เพียงแค่สร้างนโยบายรอบการใช้งาน S3 เพื่อเก็บอ็อบเจ็กต์ถาวรตลอดรอบการใช้งาน หรืออัปโหลดอ็อบเจ็กต์โดยตรงไปยังคลาสพื้นที่จัดเก็บถาวร ด้วย S3 Object Lock คุณจะสามารถนำการกำหนดวันที่มาใช้กับอ็อบเจ็กต์เพื่อป้องกันไม่ให้ถูกลบ และตรงตามกฎข้อบังคับ ต่างจากไลบรารีเทป S3 Glacier ทำให้คุณสามารถกู้คืนอ็อบเจ็กต์ที่เก็บถาวรไปแล้วโดยใช้เวลาเพียงหนึ่งนาทีสำหรับการเรียกข้อมูลแบบเร่งด่วน และ 3-5 ชั่วโมงสำหรับการเรียกข้อมูลแบบมาตรฐาน การกู้คืนข้อมูลจำนวนมากจาก S3 Glacier และการกู้คืนทั้งหมดจาก S3 Glacier Deep Archive จะเสร็จสิ้นภายใน 12 ชั่วโมง

Data Lake และการวิเคราะห์ Big Data

เร่งรัดนวัตกรรมด้วยการสร้างที่จัดเก็บข้อมูลดิบบน Amazon S3 และแยกข้อมูลเชิงลึกที่สำคัญโดยใช้เครื่องมือการสืบค้นในตัว การวิเคราะห์ และ Machine Learning เมื่อที่จัดเก็บข้อมูลดิบของคุณใหญ่ขึ้น ใช้ S3 Access Points ในการกำหนดค่าการเข้าถึงข้อมูลของคุณได้ง่ายๆ ด้วยสิทธิ์ที่เฉพาะเจาะจงสำหรับแอปพลิเคชันแต่ละรายการหรือชุดแอปพลิเคชัน คุณยังสามารถใช้ AWS Lake Formation เพื่อสร้างที่จัดเก็บข้อมูลดิบอย่างรวดเร็ว พร้อมทั้งกำหนดและบังคับใช้นโยบายความปลอดภัย การควบคุม และการตรวจสอบจากส่วนกลาง บริการจะจัดเก็บข้อมูลในฐานข้อมูลและแหล่งทรัพยากร S3 ของคุณ ย้ายไปยัง Data Lake แห่งใหม่ใน Amazon S3 และล้างและคัดแยกประเภทโดยใช้อัลกอริธึมของ Machine Learning ทรัพยากร AWS ทั้งหมดสามารถปรับขนาดขึ้นเพื่อรองรับพื้นที่จัดเก็บข้อมูลที่ขยายใหญ่ขึ้นของคุณ โดยไม่ต้องลงทุนเบื้องต้น

พื้นที่จัดเก็บบนระบบคลาวด์แบบไฮบริด

สร้างการเชื่อมต่อที่ไร้ที่ติระหว่างแอปพลิเคชันในสถานที่ทำงานและ Amazon S3 ด้วย AWS Storage Gateway เพื่อที่จะลดฟุตพรินท์ของศูนย์ข้อมูล และใช้ประโยชน์จากขนาด ความนน่าเชื่อถือ และความทนทานของ AWS รวมถึงนวัตกรรม Machine Learning และความสามารถในการวิเคราะห์ของ AWS คุณยังสามารถโอนถ่ายข้อมูลระหว่างพื้นที่จัดเก็บบนสถานที่ทำงานและ Amazon S3 ได้โดยอัตโนมัติผ่าน AWS DataSync ซึ่งสามารถถ่ายโอนข้อมูลด้วยความเร็วสูงกว่าเครื่องมือโอเพนซอร์สถึง 10 เท่า อีกวิธีในการเปิดใช้งานสภาพแวดล้อมพื้นที่เก็บข้อมูลคลาวด์แบบไฮบริดคือการร่วมงานกับผู้ให้บริการเกตเวย์จาก APN คุณยังสามารถถ่ายโอนข้อมูลได้โดยตรงไปยังและออกจาก Amazon S3 ด้วย AWS Transfer for SFTP ซึ่งเป็นบริการที่มีการจัดการอย่างเต็มรูปแบบซึ่งอนุญาตให้แลกเปลี่ยนไฟล์กับบุคคลภายนอกได้อย่างปลอดภัย
เรียนรู้เพิ่มเติมเกี่ยวกับพื้นที่จัดเก็บแบบไฮบริด AWS DataSync และ AWS Transfer for SFTP » 

ข้อมูลแอปพลิเคชันบน Cloud-native

สร้างแอปพลิเคชันบนอินเทอร์เน็ตและมือถือได้รวดเร็วและคุ้มทุนโดยใช้บริการของ AWS และ Amazon S3 เพื่อจัดเก็บข้อมูลการผลิต ด้วย Amazon S3 คุณสามารถอัปโหลดข้อมูลในปริมาณใดก็ได้และเข้าถึงได้จากทุกที่เพื่อที่จะปรับใช้แอปพลิเคชันได้อย่างรวดเร็วและเข้าถึงผู้ใช้ได้มากยิ่งขึ้น การจัดเก็บข้อมูลใน Amazon S3 ยังหมายความว่าคุณสามารถเข้าถึงเครื่องมือและบริการสำหรับ Developer ของ AWS แบบใหม่ล่าสุดสำหรับการเรียนรู้ของเครื่องและการวิเคราะห์เพื่อสรรค์สร้างนวัตกรรมและเพิ่มประสิทธิภาพให้กับแอปพลิเคชันบนระบบคลาวด์

กรณีศึกษา

Netflix จัดส่งเนื้อหาหลายพันล้านชั่วโมงจาก Amazon S3 ให้กับลูกค้าทั่วโลก Amazon S3 ยังทำหน้าที่เป็น Data Lake สำหรับโซลูชันการวิเคราะห์ Big Data ของ Netflix อีกด้วย
FINRA ใช้ Amazon S3 เพื่อป้อนและจัดเก็บข้อมูลสำหรับกิจกรรมการตลาดกว่า 75 พันล้านกิจกรรมทุกวัน และใช้ฟังก์ชัน AWS Lambda เพื่อจัดรูปแบบและตรวจสอบข้อมูลตามกฎกว่า 200 กฎ
Airbnb เก็บข้อมูลสำรองและไฟล์ที่ไม่มีการเปลี่ยนแปลงไว้บน Amazon S3 รวมถึงภาพของผู้ใช้ที่มีขนาดมากกว่า 10 เพตะไบต์ การใช้โซลูชันระบบคลาวด์มาตั้งแต่แรก ทำให้พวกเขาสามารถคิดค้นวิธีการใหม่ๆ อย่างต่อเนื่องในการวิเคราะห์ข้อมูลที่จัดเก็บไว้บน Amazon S3
GE ใช้ Amazon S3 เพื่อจัดเก็บและปกป้องข้อมูลภาพทางการแพทย์ที่สำคัญกว่าเพตะไบต์สำหรับบริการ GE Health Cloud ซึ่งเชื่อมต่อเครื่องสร้างภาพและอุปกรณ์ทางการแพทย์นับแสนเครื่อง

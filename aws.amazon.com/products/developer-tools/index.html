Use cases

Continuous integration

Continuous integration (CI) is a DevOps software development practice where developers regularly merge their code changes into a central repository, after which automated builds and tests are run.
Try AWS CodeBuild »

Continuous delivery

Continuous delivery (CD) is a software development practice where code changes are automatically prepped for a release to production. All code changes are deployed to a testing and/or a production environment after the build stage.
Try AWS CodePipeline »

Infrastructure as code

Model your applications and infrastructure as code to automate the provisioning of infrastructure services and eliminate the need for manual updates to infrastructure which can be error-prone.
Try AWS Cloud
Development Kit »

Monitoring and logging

Monitor metrics and logs to see how your applications and infrastructure performance impacts the experience of your end users. Capture, categorize, and analyze data generated by applications and infrastructure to understand the impact of code changes.
Try AWS X-Ray »
Try Amazon CloudWatch »

Amazon S3

어디서나 원하는 양의 데이터를 저장하고 검색할 수 있도록 구축된 객체 스토리지

Amazon S3 시작
추가 정보 요청

Amazon Simple Storage Service(Amazon S3)는 업계 최고의 확장성과 데이터 가용성 및 보안과 성능을 제공하는 객체 스토리지 서비스입니다. 즉, 어떤 규모 어떤 산업의 고객이든 이 서비스를 사용하여 웹 사이트, 모바일 애플리케이션, 백업 및 복원, 아카이브, 엔터프라이즈 애플리케이션, IoT 디바이스, 빅 데이터 분석 등과 같은 다양한 사용 사례에서 원하는 만큼의 데이터를 저장하고 보호할 수 있습니다. Amazon S3는 사용하기 쉬운 관리 기능을 제공하므로 특정 비즈니스, 조직 및 규정 준수 요구 사항에 따라 데이터를 조직화하고 세부적인 액세스 제어를 구성할 수 있습니다. Amazon S3는 99.999999999%의 내구성을 제공하도록 설계되었으며, 전 세계 기업의 수백만 애플리케이션을 위한 데이터를 저장합니다.

장점

업계 최고의 성능, 확장성, 가용성, 내구성

초기 투자나 리소스 공급 과정 없이도, 변화하는 요구 사항에 맞게 스토리지 리소스를 확장 및 축소할 수 있습니다. Amazon S3는 여러 시스템에 걸쳐 모든 S3 객체의 복사본을 자동으로 생성하고 저장하기 때문에99.999999999%의 데이터 내구성을 제공합니다. 즉, 필요할 때 데이터를 사용할 수 있고, 데이터를 장애, 오류, 도용 등으로부터 보호할 수 있습니다.

광범위한 비용 효율적 스토리지 클래스

다양한 데이터 액세스 수준을 해당 요율로 지원하는 S3 스토리지 클래스에 데이터를 저장하여 성능 저하 없이 비용을 절감하십시오. S3 스토리지 클래스 분석을 사용하여 액세스 패턴에 따라 더 저렴한 요금의 스토리지 클래스로 이동할 데이터를 검색할 수 있으며, 이러한 이동을 실행하는 S3 수명 주기 정책을 구성할 수 있습니다. 또한 액세스 패턴이 변하거나 액세스 패턴을 알 수 없는 데이터는, 액세스 패턴 변화에 따라 객체를 계층화하는 S3 Intelligent-Tiering에 저장하여 자동으로 비용을 절약할 수 있습니다.

타의 추종을 불허하는 보안, 규정 준수, 감사 기능

데이터를 Amazon S3에 저장하고, 암호화 기능 및 액세스 관리 도구를 통해 무단 액세스로부터 데이터를 보호하십시오. S3는 S3 퍼블릭 액세스 차단을 통해 버킷 또는 계정 수준에서 모든 객체에 대한 퍼블릭 액세스를 차단할 수 있는 유일한 객체 스토리지 서비스입니다. S3는 PCI-DSS, HIPAA/HITECH, FedRAMP, EU 데이터 보호 지침, FISMA 등과 같은 규정 준수 프로그램을 적용하여 규제 요건을 준수할 수 있도록 지원합니다. 또한 AWS는 S3 리소스에 대한 액세스 요청을 모니터링하는 다양한 감사 기능을 지원합니다.

쉬운 데이터 관리와 액세스 제어

S3는 액세스, 비용, 복제 및 데이터 보호를 관리에 대한 강력한 기능을 제공합니다. S3 액세스 포인트는 공유 데이터 세트를 사용하여 애플리케이션에 대한 특정 권한으로 데이터 액세스를 쉽게 관리할 수 있도록 지원합니다. S3 복제 기능은 리전 내 또는 다른 리전으로의 데이터 복제를 관리합니다. S3 배치 기능은 수조 개의 객체 간에 대규모 변경 사항을 관리합니다. S3는 AWS Lambda와 연동되기 때문에, 추가 인프라를 관리하지 않고도 활동을 기록하고 알림을 정의하며 워크플로를 자동화할 수 있습니다.

분석을 위한 쿼리 인 플레이스 서비스

AWS의 쿼리 인 플레이스 서비스를 사용하여 S3 객체 및 AWS의 다른 데이터 세트에 대해 빅 데이터 분석을 실행하십시오. Amazon Athena를 사용하여 표준 SQL 표현식을 통해 S3 데이터를 쿼리하고, Amazon Redshift Spectrum을 사용하여 AWS 데이터 웨어하우스 및 S3 리소스에 저장된 데이터를 분석하십시오. 또한 S3 Select를 사용하면 전체 객체가 아닌 객체 데이터의 일부를 검색함으로써 쿼리 성능을 최대 400% 높일 수 있습니다.

지원성이 가장 높은 클라우드 스토리지 서비스

최대의 기술 및 컨설팅 클라우드 서비스 공급업체 커뮤니티인 AWS 파트너 네트워크(APN)의 파트너와 협력하여 Amazon S3에 데이터를 저장하고 보호하십시오. APN은 데이터를 Amazon S3에 전송하는 마이그레이션 파트너와, 기본 스토리지, 백업 및 복원, 아카이브 및 재해 복구를 위한 S3 통합 솔루션을 제공하는 스토리지 파트너를 공식으로 승인합니다. 250개 이상의 스토리지 제품을 제공하는 AWS Marketplace에서 AWS 통합 솔루션을 직접 구입할 수도 있습니다.

작동 방식 - S3 액세스 포인트

Amazon S3 액세스 포인트는 S3의 공유 데이터 세트를 사용하여 애플리케이션에 대한 대규모 데이터 액세스 관리를 간소화합니다. S3 액세스 포인트를 사용하여 버킷당 수백 개의 액세스 포인트를 쉽게 생성할 수 있습니다. 즉, 공유 데이터 세트에 대한 액세스를 프로비저닝하는 새로운 방식을 나타냅니다. 액세스 포인트는 고유한 호스트 이름과 액세스 포인트를 통해 생성된 모든 요청에 대해 특정 권한 및 네트워크 제어를 적용하는 액세스 정책을 사용하여 버킷으로의 사용자 지정된 경로를 제공합니다.

사용 사례

백업 및 복원

Amazon S3 및 S3 Glacier, Amazon EFS, Amazon EBS 등의 기타 AWS 서비스를 통해 확장 가능하고 내구성이 뛰어나며 안전한 백업 및 복원 솔루션을 구축하여 기존 온프레미스 기능을 강화하거나 대체하십시오. AWS 및 APN 파트너가 복구 목표 시간(RTO), 복구 목표 시점(RPO) 및 규정 준수 요건을 충족할 수 있도록 도와드리겠습니다. AWS를 사용하면, AWS 클라우드의 데이터를 백업하거나, 하이브리드 스토리지 서비스인 AWS Storage Gateway를 사용하여 온프레미스 데이터의 백업을 AWS로 전송할 수 있습니다.

재해 복구(DR)

AWS 클라우드 또는 온프레미스 환경에서 실행되는 중요한 데이터, 애플리케이션 및 IT 시스템을, 별도의 물리적 사이트를 유지하는 추가 비용 없이 보호할 수 있습니다. Amazon S3 스토리지, S3 리전 간 복제, 기타 AWS 컴퓨팅, 네트워킹, 데이터베이스 서비스를 사용하면 재해 복구(DR) 아키텍처를 구성하여 자연 재해나 시스템 고장 및 작업자 실수로 인해 발생하는 운영 중지를 신속하고도 손쉽게 복구할 수 있습니다.

아카이브

물리적 인프라에서 전환하여 S3 Glacier 및 S3 Glacier Deep Archive로 데이터를 아카이브하십시오. 이러한 S3 스토리지 클래스는 가장 저렴한 요금으로 장기간 객체를 보관합니다. 수명 주기 동안 객체를 아카이브하거나 아카이브 스토리지 클래스에 직접 객체를 업로드하는 S3 수명 주기 정책을 생성하면 됩니다. S3 객체 잠금을 사용하면, 객체에 보존 날짜를 적용하여 객체가 삭제되지 않도록 보호하고 규정 준수 요건을 준수할 수 있습니다. 테이프 라이브러리와 달리 S3 Glacier는 신속 검색의 경우 아카이브된 객체를 1분 내에 복원하고, 표준 검색의 경우 3-5시간 내에 복원할 수 있습니다. S3 Glacier의 대량 데이터 복원 및 S3 Glacier Deep Archive의 모든 복원은 12시간 내에 완료됩니다.

데이터 레이크와 빅 데이터 분석

Amazon S3에 데이터 레이크를 구축하여 혁신을 가속화하고, 쿼리 인 플레이스, 분석 및 기계 학습 도구를 사용하여 가치 있는 통찰력을 확보하십시오. 데이터 레이크 규모 확장에 따라 애플리케이션 또는 애플리케이션 세트에 대해 특정 권한이 있는 S3 액세스 포인트를 사용하여 데이터에 대한 액세스를 쉽게 구성할 수 있습니다. 또한 AWS Lake Formation을 사용하면 데이터 레이크를 신속하게 생성할 수 있으며, 보안, 거버넌스 및 감사 정책을 중앙에서 정의하고 적용할 수 있습니다. 이 서비스는 데이터베이스와 S3 리소스에서 데이터를 수집하여 Amazon S3의 새로운 데이터 레이크로 이동한 후 기계 학습 알고리즘을 사용하여 데이터를 정리하고 분류합니다. 모든 AWS 리소스는 초기 투자 없이 데이터 스토어가 증가함에 따라 확장할 수 있습니다.

하이브리드 클라우드 스토리지

데이터 센터 설치 공간을 줄이고 AWS의 확장성, 안정성 및 내구성과 AWS의 혁신적인 기계 학습 및 분석 기능을 활용하려면 AWS Storage Gateway를 사용하여 온프레미스 애플리케이션과 Amazon S3를 원활하게 연결하십시오. 오픈 소스 도구보다 최대 10배 빠른 속도로 데이터를 전송할 수 있는 AWS DataSync를 사용하여 온프레미스 스토리지와 AWS(Amazon S3 포함) 간에 데이터 전송을 자동화할 수도 있습니다. 하이브리드 클라우드 스토리지 환경을 지원하는 또 다른 방법은 APN의 게이트웨이 공급자와 협력하는 것입니다. 제3자와 파일을 안전하게 교환할 수 있도록 지원하는 종합 관리형 서비스인 AWS Transfer for SFTP를 사용하여 Amazon S3로 또는 Amazon S3에서 직접 파일을 전송할 수도 있습니다.

클라우드 네이티브 애플리케이션 데이터

AWS 서비스와 Amazon S3를 사용하여 프로덕션 데이터를 저장하여 빠르고 비용 효율적인 모바일 및 인터넷 기반 애플리케이션을 구축하십시오. Amazon S3를 사용하면 용량에 상관없이 데이터를 업로드하고 어디에서든 액세스하여 애플리케이션을 더 빠르게 배포하고 더 많은 사용자가 액세스하게 할 수 있습니다. 또한 Amazon S3에 데이터를 저장하면 기계 학습 및 분석을 위한 최신 AWS 개발자 툴과 서비스에 액세스하여 클라우드 네이티브 애플리케이션을 혁신하고 최적화할 수 있습니다.

사례 연구

Netflix는 Amazon S3의 수십억 시간에 달하는 콘텐츠를 전 세계 고객에게 제공합니다. 또한, Amazon S3는 빅 데이터 분석 솔루션을 위한 데이터 레이크로 사용됩니다.
FINRA는 Amazon S3를 사용하여 매일 750억 개 이상의 시장 이벤트에 대한 데이터를 수집 및 저장하고, AWS Lambda 함수를 사용하여 200개 이상의 규칙에 대해 데이터를 포맷하고 검증합니다.
Airbnb는 10페타바이트가 넘는 사용자 사진을 비롯하여 백업 데이터와 정적 파일을 Amazon S3에 저장합니다. 클라우드에서 탄생한 솔루션인 Airbnb는 Amazon S3에 저장된 데이터를 분석하는 새로운 방법을 끊임없이 혁신하고 있습니다.
